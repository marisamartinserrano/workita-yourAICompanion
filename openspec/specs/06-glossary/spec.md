# Spec: 06 — Glossary

## Overview

The Glossary provides candidates with a comprehensive, searchable reference of industry-specific and role-specific terms. Glossaries are generated by AI from job descriptions and linked to specific candidatures, giving users the language they need to communicate confidently in interviews and applications.

## Requirements

### Functional Requirements

**Generation**
- FR-01: The user MUST be able to generate a glossary by providing:
  - Target Role (required)
  - Seniority (optional)
  - Industry (optional)
  - Company (optional)
  - Job description text or URL (optional, enriches results)
- FR-02: The AI MUST generate between 20 and 50 terms per request, calibrated to the inputs.
- FR-03: Each term MUST include:
  - Term name
  - Clear definition
  - Contextual explanation (why this term matters for the role/industry)
- FR-04: A glossary MUST be automatically generated and saved when a new candidature is created (see Spec 03 — New Candidature).

**Storage & Access**
- FR-05: Generated glossaries MUST be persisted in the database and linked to the associated candidature (if applicable).
- FR-06: The user MUST be able to access any saved glossary at any time from the Glossary section.
- FR-07: The user MUST be able to search glossary terms by keyword.
- FR-08: The user MUST be able to filter glossaries by candidature, role, or industry.
- FR-09: The glossary MUST be continuously enriched as new candidatures are created.

**Display**
- FR-10: Terms MUST be displayed in a card or list format showing: term, definition, and context.
- FR-11: The search filter MUST match against both term name and definition text.

### Non-Functional Requirements

- NFR-01: Glossary generation MUST complete within 10 seconds.
- NFR-02: Glossaries MUST be indexed for fast search (client-side filter acceptable for ≤500 terms).

## Scenarios

### Scenario 1: Generating a standalone glossary
**Given** the user is on the Glossary page  
**When** they enter "Product Manager", "Senior", "Fintech" and click "Generate Glossary"  
**Then** the AI returns 20–50 terms with definitions and context, displayed as cards

### Scenario 2: Auto-generated glossary from a candidature
**Given** a new candidature has been created for a Data Engineer role  
**When** the user opens the Glossary section  
**Then** the terms generated from that job description are visible and linked to the candidature

### Scenario 3: Searching the glossary
**Given** the glossary has 35 terms  
**When** the user types "ATS" in the search box  
**Then** only terms containing "ATS" in their name or definition are shown

### Scenario 4: Accessing a saved glossary
**Given** the user generated a glossary 3 days ago for a Marketing Manager role  
**When** they return to the Glossary section  
**Then** the previously generated glossary is still available and searchable

## Acceptance Criteria

- [ ] Role, seniority, industry, company, and job description inputs are available
- [ ] AI generates 20–50 terms per request
- [ ] Each term shows: name, definition, and contextual explanation
- [ ] Glossary is auto-generated when a new candidature is created
- [ ] Glossaries are saved to the database
- [ ] Saved glossaries are accessible at any time
- [ ] Search filters by keyword across term name and definition
- [ ] Terms can be filtered by candidature, role, or industry
