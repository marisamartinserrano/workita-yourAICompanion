# Spec: 09 — LinkedIn Recommendations

## Overview

The LinkedIn Recommendations feature provides candidates with specific, actionable advice to improve their LinkedIn profile. Recommendations are generated by AI using the candidate's CV content and LinkedIn algorithm best practices. A collapsible explainer helps candidates understand how LinkedIn works before acting on recommendations. Recommendations are always generated — even when no LinkedIn URL is provided — though they are more targeted when a profile URL is available.

## Requirements

### Functional Requirements

**LinkedIn Profile Recommendations**
- FR-01: The system SHALL generate specific, actionable recommendations to improve the candidate's LinkedIn profile, based on:
  - CV content
  - LinkedIn algorithm best practices (profile summary, skills section, headline, consistency with CV, ATS filtering)
- FR-02: Each recommendation SHALL include a rationale explaining why the change matters.
- FR-03: Recommendations SHALL be displayed on the LinkedIn tab of the onboarding/profile page after CV analysis completes.

**LinkedIn Algorithm Explainer**
- FR-04: The LinkedIn tab SHALL display a collapsible "How LinkedIn works for candidates" section above the recommendations list.
- FR-05: The explainer SHALL cover: profile completeness, keyword matching, and recruiter search behaviour.
- FR-06: The section SHALL be collapsed by default and expandable on user interaction.

**Recommendations Without LinkedIn URL**
- FR-07: If no LinkedIn URL is saved, the system SHALL still generate LinkedIn recommendations based on CV content alone.
- FR-08: When recommendations are generated without a LinkedIn URL, an inline note SHALL inform the user that results would be more specific with a LinkedIn URL provided.

### Non-Functional Requirements

- NFR-01: LinkedIn recommendations MUST be scoped to the authenticated user and never exposed to other users.

## Scenarios

### Scenario 1: Recommendations generated after analysis
**Given** CV analysis has completed and a LinkedIn URL is saved  
**When** the user views the LinkedIn tab  
**Then** they see a list of specific improvement recommendations, each with a rationale explaining why the change matters

### Scenario 2: Explainer visible before recommendations
**Given** the user views the LinkedIn tab after analysis  
**When** the tab renders  
**Then** they see a collapsible "How LinkedIn works for candidates" section above the recommendations list

### Scenario 3: Recommendations work without URL
**Given** the user triggers analysis without a LinkedIn URL saved  
**When** the LinkedIn tab is viewed  
**Then** LinkedIn recommendations are still generated from CV content, with a note that results would be more specific with a LinkedIn URL

## Acceptance Criteria

- [ ] LinkedIn recommendations list is shown on the LinkedIn tab after analysis
- [ ] Each recommendation includes a rationale
- [ ] Collapsible "How LinkedIn works for candidates" explainer is shown above recommendations (collapsed by default)
- [ ] Recommendations are generated even when no LinkedIn URL is provided
- [ ] When no LinkedIn URL is provided, an inline note about improved specificity is shown
- [ ] Recommendations are scoped to the authenticated user
